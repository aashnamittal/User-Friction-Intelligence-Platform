# User Friction Intelligence Platform (UFIP)

UFIP is a production-grade, AI-powered customer friction detection and automated revenue recovery console. It monitors user clickstream telemetry, computes experience friction scores, cohorts users into behavioral segments via unsupervised machine learning, and uses a local LLM to generate recovery action plans for frustrated users.

## Core Components

### 1. Database & Schema Model (MySQL)
All behavioral logs and computed scoring records are stored in a MySQL database.
* **Tables**:
  * `users`: Customer profiles.
  * `sessions`: Navigation sessions mapped to user IDs, devices, and locations.
  * `events`: Granular clickstream logs (page views, clicks, rage clicks, errors, network latencies, checkouts, and checkout abandonment).
  * `friction_scores`: Calculated telemetry friction metric (0–100).
  * `user_cohorts`: Unsupervised ML classification segments.
  * `recovery_recommendations`: Copied drafts generated by LLM for high-friction accounts.
* **SQL Aggregation View (`v_user_features`)**:
  Aggregates raw telemetry events per user to compile metrics like total sessions, rage clicks count, network error events, average response latency, and checkout abandonments.

### 2. Telemetry Friction Scorer
Calculates user experience health from $0.0$ (smooth) to $100.0$ (severely blocked) using a weighted telemetry heuristic:

$$\text{Friction Score} = \min\left(\text{Rage Clicks Friction} + \text{Error Friction} + \text{Latency Friction} + \text{Checkout Abandonment Friction},\, 100.0\right)$$

* **Rage Clicks**: Each rage click adds $10.0$ points (capped at $40.0$).
* **System Errors**: Each error adds $8.0$ points (capped at $30.0$).
* **Network Latency**: Latency above $500\text{ ms}$ adds $1.0$ point per $100\text{ ms}$ (capped at $20.0$).
* **Checkout Abandonment**: Abandoning a checkout page adds $30.0$ points.

### 3. Unsupervised User Cohorting (KMeans & MLflow)
A **KMeans** clustering model segments users into four behavioral categories:
* 🟢 **Happy Active Users**: Low friction, completed checkouts, normal latencies.
* 🔴 **Frustrated Checkout Users**: High rage clicks or payment form abandonments.
* 🟡 **Slow Network Users**: Average latencies exceeding $1000\text{ ms}$.
* 🟠 **Error Prone Users**: High error rates but not checkout-blocked.

Every clustering run is tracked in **MLflow** (logging inertia, silhouette score metrics, hyperparameters, and the serialized `model.pkl` artifact).

### 4. Local AI Recovery Copywriter (Ollama & Llama 3.2)
For users with friction scores exceeding a configurable threshold (default: $30/100$), a local **Ollama** server running **Llama 3.2** generates bespoke recovery actions (e.g. customized emails, apology codes, bug tickets). If the local model is offline, the system seamlessly uses a robust template fallback.

### 5. Control Center & Dashboard (Streamlit)
An interactive management console split into three tabs:
* **📊 Cohort Analysis**: Displays high-level KPIs, cohort segment counts, and average feature statistics.
* **🔍 User Friction Lookup**: Offers detailed profile drilldowns, full clickstream timelines styled by event type, and on-demand AI recommendation regeneration.
* **⚙️ Control Panel**: Monitors service connections (MySQL, Ollama) and lets administrators trigger the Prefect ETL flow.

## Installation & Setup

Ensure you have **Python 3.10+** and **Homebrew** installed.

### 1. Start Services
Ensure local services are up:
```bash
# Start MySQL (Port 3307)
make start-db

# Start Ollama Local LLM
make start-ollama
```

### 2. Setup Virtual Environment
Install virtual environments and editable project dependencies:
```bash
make setup
```

### 3. Restore Database Schema & Data
Import the pre-populated database dump containing schemas, seeds, clusters, and recommendations:
```bash
mysql -h 127.0.0.1 -P 3307 -u root -e "CREATE DATABASE IF NOT EXISTS ufip;"
mysql -h 127.0.0.1 -P 3307 -u root ufip < database/ufip_dump.sql
```

### 4. Start MLflow and Dashboard Servers
Run tracking services in separate terminal windows:
```bash
# Terminal 1: Start MLflow server
.venv/bin/mlflow server --host 127.0.0.1 --port 5001

# Terminal 2: Start Streamlit app
make start-dashboard
```

Open your browser to:
* **Streamlit Console**: [http://localhost:8501](http://localhost:8501)
* **MLflow Experiment Panel**: [http://localhost:5001](http://localhost:5001)
